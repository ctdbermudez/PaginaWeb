Arquitectura Cliente/Servidor y Desarrollo Web

La arquitectura cliente/servidor es la base de la programación web dinámica. El cliente (navegador) solicita información y el servidor procesa los datos y devuelve la respuesta. Este modelo permite la centralización de datos, la escalabilidad y la seguridad en los sistemas web (Blog ingranetnetborking 2019; Blog La Arquitectura,2021).

Diseño Centrado en el Usuario (DCU)

El diseño centrado en el usuario busca que las aplicaciones web sean claras, accesibles y útiles para las personas. Esto implica interfaces fáciles de usar, retroalimentación inmediata y prevención de errores. El DCU asegura que las respuestas del servidor se presenten de forma comprensible y eficiente.

Investigación en el Entorno y Requisitos

Investigar el contexto donde se usará una aplicación web es esencial para definir requisitos y especificaciones técnicas. Si se desconoce la realidad del entorno conectividad y dispositivos, se corre el riesgo de construir un sistema poco funcional. La investigación permite soluciones adaptadas, seguras y sostenibles en el tiempo.
